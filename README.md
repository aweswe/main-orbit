# Welcome to Orbit 🪐

**Orbit** is an advanced AI-powered web development environment inspired by Bolt.new and Lovable. What started as a standalone AI UI builder has evolved into an enterprise-ready, auto-healing WebContainer environment capable of generating complex, multi-file React/Vite applications.

## 🚀 The Path to 100% Error-Free Code Generation

Orbit features a sophisticated **Auto-Healing Pipeline** and **Pre-Flight Validation Engine** designed to anticipate and instantly resolve LLM hallucinations before the user ever sees a broken preview. 

Here's how Orbit ensures near-perfect generation stability:

### 🛡️ 1. Determinism Constraints & Pre-Flight Gate
Before any code generated by the LLM touches the virtual file system, it passes through the Pre-Flight Validation Engine (`pre-flight.ts`):
- **JSX Extension Matching:** Automatically renames `.ts` files to `.tsx` if JSX is detected, updating all relative importers across the project simultaneously.
- **Regex & String Sanitization:** Intercepts and corrects double-scaped regex (`\\/`) and template literal parsing failures inherently common to JSON-streamed LLM responses.
- **Hallucinated Import Protection:** Ships with an `IMPORT_HALLUCINATION_REGISTRY` that actively scans and strips structurally invalid imports (like the LLM inventing `import { Netflix } from 'lucide-react'`).

### 📦 2. Pre-Boot Dependency Auditor
The LLM often generates valid component imports but forgets to add the third-party libraries to `package.json`. 
- Before starting the Vite Dev Server, Orbit parses the AST of every generated file (`dependency-auditor.ts`).
- It extracts all bare module specifiers (e.g., `framer-motion`, `@dnd-kit/core`).
- If an import is missing from `package.json`, Orbit automatically patches the `package.json` file and triggers a silent background `npm install` before the dev server is allowed to boot.
- Applies strict version pinning to prevent peer-dependency conflicts (like `@dnd-kit/core` vs `@dnd-kit/sortable`).

### 🤖 3. The 3-Tiered Auto-Healer
If an error bypasses the pre-flight checks, Orbit's multi-layered Auto-Healer catches the crash and silently dispatches a structural fix request back to the LLM:
1. **Terminal Healing:** The `ActionRunner` streams all Vite Dev Server output to a custom `error-detector.ts`. Build failures, TS errors, and ESLint warnings are instantly parsed and healed.
2. **NPM Installation Healing:** `npm install` streams are actively monitored. If the LLM invents a package version (yielding an `ETARGET` error), the Auto-Healer patches `package.json` and automatically reboots the environment.
3. **Browser Runtime Interception:** A hidden React Error Boundary wraps the entire generated application iframe. If the code compiles but crashes the React tree at runtime (e.g. `TypeError: Cannot read properties of undefined`), the stack trace is beamed out of the iframe (`window.parent.postMessage`) and piped directly into the Auto-Healer. 

### 🧠 4. AST-Based Context Engine (RAG)
For massive codebase additions, simply dumping files into the prompt window blows out the LLM API token limits. Orbit uses a custom `GraphBuilder` (`workspace-context.ts`) to semantically score dependencies and imports, supplying the LLM with just the *signatures* of Tier 2 and Tier 3 dependencies to keep generation highly focused and deterministically accurate.

## 🛠️ Next Steps & Future Sessions
To reach our goal of **100% Error-Free generations**, upcoming enhancements include:
1. **Intelligent UI/UX Planners & Prompt Refinement:** Injecting an intermediate "Planner" AI step that digests vague user prompts, understands current system constraints, maps out the UI/UX architecture beforehand, and feeds a heavily reasoned, deterministic spec sheet to the code-generation engine.
2. **On-Device & Multi-Model Orchestration:** Integrating **Gemini Nano**, Pro models, and specialized APIs (like Banana) to run specialized tasks (like image generation) natively or optimally. 
3. **Automated Asset & Image Sourcing:** Teaching the Dependency Auditor to recognize when a UI needs stock photography, automatically generating or fetching images via API, installing necessary libraries, and placing them perfectly into the WebContainer.
4. **Self-Correcting UI Tests:** Injecting Playwright/Puppeteer into the WebContainer to allow the LLM to visually test routing flows and interactive elements.
5. **Database Schema Auto-Migrations:** Intercepting Supabase/Postgres API errors and automatically generating SQL migrations when the LLM hallucinates new columns.
6. **Context Graph Caching:** Storing the AST graphs globally to allow instant sub-100ms context rebuilds on large Mono-repos.

## What technologies are used for this project?

This IDE environment is built with:
- **WebContainers API** (by StackBlitz)
- **Vite**
- **TypeScript** & **React**
- **shadcn-ui** & **Tailwind CSS**
- **LLM APIs** (Groq, OpenAI, Anthropic) 

## How to Run Locally

Clone the repository, and boot the environment:

```sh
# Step 1: Install dependencies
npm i

# Step 2: Start the development server
npm run dev
```
